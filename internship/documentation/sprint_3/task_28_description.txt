Любое ИТ решение трудно себе представить без процесса
валидации входных данных. Валидация входных данных становится
очень актуальной, когда с приложением взаимодействует человек.
Человеку по своей природе свойственно ошибаться. Он запросто
может оставить важное поле не заполненным, или вписать в поле
вместо цифры буквы. Для выявления таких ситуаций и нужна
валидация входных данных в ИТ решениях.

Нам необходимо реализовать валидацию входных данных.
Это не самое простое задание. Ментор создал прототип
(возможно не до конца правильно работающее решение)
кода валидации.

Ваша задача: ознакомиться с прототипом валидации и понять
как он работает. Если в процессе изучения кода у вас возникнут
вопросы, задавайте их!

Внедрите код валидации предложенный ментором в свой проект!

PS: быстро найти все изменения сделанные в текущем задании
можно в IDEA (закладка слева) выбрав папку текущего задания (например task_10)
и папку предыдущего задания (например task_9) и в сплывающем меню выбрав пункт
Compare Directories (Ctrl + D).

PS: не забудьте по окончанию выполнения шага создать и заполнить
файл /documentation/Task_x_description.txt. Формат заполнения этого
файла описан в task_1 в файле Learning_reflection.txt.
