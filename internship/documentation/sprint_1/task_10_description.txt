Ваша задача: в классе TravelCalculatePremiumRequest создать
- дефолтный конструктор (конструктор без параметров);
- конструктор для задания начальных значений всем свойствам;
- get/set методы для всех свойств класса.

При выполнении каждого задания вносите изменения только в
файлы лежащие в вашей папке. Не вносите изменения в папки
других участников проекта.

После выполнения задания каждого шага вам необходимо закомитить
сделанные вами изменения в локальный Git репозиторий и запушить
их в глобальный репозиторий проекта на GitHub.

При комите изменений в локальный репозиторий в Git commit message
указывайте сообщение следующего формата: STEP XXX - DONE.
Например, STEP 001 - DONE, STEP 002 - DONE, STEP 100 - DONE и так далее.
По комит сообщению будет легко понять работу над каким шагом вы закончили.

Если вы все таки закомитите в репозиторий код с ошибками, вы должны
создать новый комит с исправлениями и указать в его сообщении:
STEP 001 - FIX.

При пуше изменений в глобальный репозиторий на GitHub выбирайте
опцию Rebase. Найдите в интернете, что означают опции Git Rebase и Merge.

https://habr.com/ru/articles/432420/

PS: запушить ваш код в глобальный репозиторий на GitHub вы сможете,
если вашему пользователю будут выданы права + вы их подтвердите.
GitHub для подтверждения доступа к репозиторию присылает емейл с
линком для подтверждения получения прав.

PS: быстро найти все изменения сделанные в текущем задании
можно в IDEA (закладка слева) выбрав папку текущего задания (например task_10)
и папку предыдущего задания (например task_9) и в сплывающем меню выбрав пункт
Compare Directories (Ctrl + D).
